MVP Blueprint: Local AI Agency Command Center1. Project OverviewA cross-platform Flutter application (iOS/Android/Desktop) that acts as a local-first AI Agent.Core Intelligence: LM Studio (Local LLM Server).Orchestration: langchain_dart.Primary Function: A "Private AI Employee" that can read local files, find business leads online, log data to Google Sheets, and notify the user of important events.2. Tech Stack & DependenciesFrontendFramework: FlutterState Management: GetX (Reactive architecture)UI Components: flutter_markdown, file_picker, lucide_iconsAI & DataLLM Orchestrator: langchain, langchain_openai (Configured for LM Studio)Local Database/Vector Store: objectbox or sqfliteText Splitting: langchain_text_splittersIntegrationsGoogle Sheets: googleapis, google_sign_inSearch API: tavily_dart (or http for Serper.dev)Notifications: flutter_local_notifications3. Architecture (Clean Architecture)Follow this folder structure for the implementation:lib/core/: Constants, Theme, and API Configuration (Base URL: http://localhost:1234/v1).lib/data/:repositories/: Implementations of LLM and Sheets logic.services/: GoogleAuthService, NotificationService, VectorStoreService.tools/: LangChain tool wrappers for Google Sheets and Web Search.lib/domain/: Entities (Message, Lead) and Repository interfaces.lib/presentation/:chat/: Chat screen with streaming response support.leads/: A dashboard to view/manage leads from Google Sheets.settings/: LM Studio connection status and Google Login.4. Key Logic & Agent ToolsTool 1: Google Sheets ManagerFunction: append_to_sheet(List<dynamic> row)Logic: Uses SheetsApi to append a new row to a specific spreadsheetId.Purpose: Save leads found online or log work activity.Tool 2: Lead Hunter (Web Search)Function: search_leads(String query)Logic: Calls Tavily/Serper API. Filters results for business contact info and specific "marketing gaps" (e.g., "No website").Purpose: Finding new clients autonomously.Tool 3: System NotifierFunction: send_alert(String message)Logic: Triggers flutter_local_notifications with a high-priority channel.Purpose: Alert the user when a high-value lead is found or a task is finished.Tool 4: Local RAG (Knowledge Base)Logic: Uses RecursiveCharacterTextSplitter to chunk uploaded PDFs and stores them locally. The Agent queries this context before answering.5. Implementation Instructions for AI AssistantStep 1: ConnectivityConfigure the ChatOpenAI client in Dart:final llm = ChatOpenAI(
  apiKey: 'not-needed',
  baseUrl: 'http://localhost:1234/v1', // 10.0.2.2:1234 for Android Emulator
  defaultOptions: const ChatOpenAIOptions(
    model: 'local-model',
    temperature: 0.7,
  ),
);
Step 2: Agent PromptingSet the system prompt to:"You are a Senior Agency Growth Agent. You help the user find leads and manage data.Use Search to find local business leads.Use Google Sheets to save leads you find.Use Notifications to alert the user about high-value findings.Use the Knowledge Base tool for information from local documents."Step 3: UI RequirementsImplement a ListView.builder for chat that supports "Thinking" states.Implement a FilePicker button that triggers a background "Ingestion" process.Show a "Success" snackbar/notification when the LLM successfully writes to Google Sheets.6. Critical NotesCORS: Ensure LM Studio has CORS enabled.OAuth: The user must sign in via Google to get the authHeaders required for the Sheets API.Streaming: Use llm.stream(prompt) to ensure the Flutter UI updates token-by-token for a "ChatGPT-like" feel.
